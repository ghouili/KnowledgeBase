Comment aborder un TPHow to approach a lab

Chaque TP suit la même logique professionnelle : on prépare, on agit avec méthode, on teste, puis on documente. Lisez l'objectif et le barème avant de commencer.Every lab follows the same professional logic: prepare, act methodically, test, then document. Read the objective and rubric before starting.

Règles d'or : ① Sécurité d'abord (alimentation débranchée, bracelet antistatique pour le matériel ; instantané/point de restauration pour le logiciel). ② Une action à la fois. ③ Notez tout (commande, version, capture). ④ Le livrable est obligatoire pour la notation.Golden rules: ① Safety first (power unplugged, anti-static strap for hardware; snapshot/restore point for software). ② One change at a time. ③ Record everything (command, version, screenshot). ④ The deliverable is required for grading.
Astuce de rendu : nommez votre livrable TP<n>_Nom_Prenom.pdf et datez chaque capture d'écran. Un travail tracé est un travail crédible.Submission tip: name your deliverable Lab<n>_Last_First.pdf and date every screenshot. Traceable work is credible work.

Inventaire matériel sans démontageHardware inventory without disassembly

⏱ 1 hIndividuelSoloDébutantBeginner

ObjectifObjective

Identifier les caractéristiques réelles de votre PC (CPU, RAM, disque, GPU) à l'aide des outils du système.Identify your PC's real specs (CPU, RAM, disk, GPU) using built-in tools.

Matériel / prérequisMaterials / prerequisites

Un PC sous Windows ou Linux. Aucun démontage.A Windows or Linux PC. No disassembly.

Étapes guidéesGuided steps

  1. Ouvrez les infos système : Windows → Paramètres → Système → À propos, ou Win + R puis msinfo32.Open system info: Windows → Settings → System → About, or Win + R then msinfo32.
  2. Relevez le CPU (modèle, fréquence, nombre de cœurs) et la RAM (quantité et type DDR).Note the CPU (model, clock, cores) and RAM (amount and DDR type).
  3. Identifiez le disque (HDD/SSD, capacité) et le GPU.Identify the disk (HDD/SSD, capacity) and the GPU.
  4. Vérifiez en ligne de commande puis comparez.Verify on the command line, then compare.
# Windows (cmd / PowerShell)
wmic cpu get name,NumberOfCores,MaxClockSpeed
wmic memorychip get Capacity,Speed,MemoryType
wmic diskdrive get Model,Size

# Linux
lscpu | grep -E "Model name|CPU\(s\)"
free -h
lsblk -d -o NAME,SIZE,ROTA   # ROTA=1 HDD, 0 SSD
Astuce : sous Windows, des outils gratuits comme CPU-Z ou Speccy donnent le type exact de RAM (DDR3/4/5) et la fréquence réelle.Tip: on Windows, free tools like CPU-Z or Speccy reveal the exact RAM type (DDR3/4/5) and real frequency.

Une fiche d'inventaire (tableau) avec captures à l'appui.An inventory sheet (table) backed by screenshots.

TP 1.2

Démontage / remontage en sécuritéSafe disassembly / reassembly

⏱ 2 hBinômePairsPratiqueHands-on

ObjectifObjective

Démonter puis remonter une unité centrale en respectant les règles de sécurité, et identifier visuellement les composants.Disassemble then reassemble a desktop unit following safety rules, and visually identify the components.

Matériel / prérequisMaterials / prerequisites

Unité centrale d'atelier, tournevis cruciforme, bracelet antistatique, plan de travail dégagé.A lab desktop unit, Phillips screwdriver, anti-static wrist strap, clear workspace.

Étapes guidéesGuided steps

  1. Sécurité : éteindre, débrancher l'alimentation, attendre 30 s, mettre le bracelet antistatique relié au châssis.Safety: power off, unplug, wait 30 s, put on the anti-static strap clipped to the chassis.
  2. Ouvrir le panneau latéral. Photographier l'intérieur avant toute manipulation (référence pour le remontage).Open the side panel. Photograph the inside before touching anything (reference for reassembly).
  3. Repérer et déposer : barrette(s) de RAM (clips), disque (SSD/HDD), une carte d'extension (PCIe).Locate and remove: RAM stick(s) (clips), drive (SSD/HDD), one expansion card (PCIe).
  4. Identifier sans les déposer : socket CPU, chipset, ports SATA, slot M.2, connecteurs d'alimentation.Identify without removing: CPU socket, chipset, SATA ports, M.2 slot, power connectors.
  5. Remonter dans l'ordre inverse, refermer, rebrancher, démarrer et vérifier le POST.Reassemble in reverse order, close, reconnect, boot and check the POST.
⚠ Sécurité : ne jamais forcer une barrette ou une carte ; manipuler la RAM par les bords ; ne pas toucher les contacts dorés.⚠ Safety: never force a stick or card; hold RAM by the edges; don't touch the gold contacts.

TP 2.1

Exploration du BIOS / UEFIExploring the BIOS / UEFI

⏱ 1 hIndividuelSoloDébutantBeginner

ObjectifObjective

Accéder au firmware d'amorçage et repérer les réglages clés (ordre de boot, virtualisation, Secure Boot, heure).Access the boot firmware and locate the key settings (boot order, virtualisation, Secure Boot, time).

Étapes guidéesGuided steps

  1. Redémarrer et appuyer sur la touche du firmware : F2, F10, F12 ou Suppr selon le constructeur.Reboot and press the firmware key: F2, F10, F12 or Del depending on the maker.
  2. Localiser l'ordre de démarrage (Boot Order) et noter le 1er périphérique amorçable.Find the Boot Order and note the first bootable device.
  3. Repérer la virtualisation (Intel VT-x / AMD-V) — utile pour le TP 2.2.Locate virtualisation (Intel VT-x / AMD-V) — needed for Lab 2.2.
  4. Observer Secure Boot, le mode UEFI vs Legacy, et l'heure (pile CMOS).Observe Secure Boot, UEFI vs Legacy mode, and the time (CMOS battery).
  5. Quitter sans enregistrer (Exit without saving) si vous n'aviez rien à changer.Exit without saving if you had nothing to change.
⚠ Prudence : ne modifiez pas l'ordre de boot ou Secure Boot sur un poste de production sans consigne. En cas de doute, quittez sans enregistrer.⚠ Caution: don't change boot order or Secure Boot on a production machine without instruction. When in doubt, exit without saving.

TP 2.3

Gestion de fichiers en ligne de commande (DOS & Bash)File management on the command line (DOS & Bash)

⏱ 1 h 30IndividuelSoloPratiqueHands-on

ObjectifObjective

Reproduire et manipuler une arborescence de fichiers en DOS puis en Bash, et établir l'équivalence des commandes.Reproduce and manipulate a file tree in DOS then Bash, and map the command equivalences.

Étapes guidéesGuided steps

  1. Créer l'arborescence Cours/TP/Notes, y créer des fichiers, en copier, déplacer, renommer puis supprimer.Create the tree Cours/TP/Notes, create files in it, copy, move, rename then delete them.
  2. Réaliser les mêmes opérations dans les deux environnements et noter chaque commande.Perform the same operations in both environments and record each command.
  3. Afficher l'arborescence finale (tree / ls -R).Show the final tree (tree / ls -R).
# DOS (cmd)
md Cours\TP\Notes
echo cours > Cours\TP\notes.txt
copy Cours\TP\notes.txt Cours\TP\Notes\copie.txt
move Cours\TP\Notes\copie.txt Cours\
ren Cours\copie.txt final.txt
tree /F

# Bash (Linux)
mkdir -p Cours/TP/Notes
echo cours > Cours/TP/notes.txt
cp Cours/TP/notes.txt Cours/TP/Notes/copie.txt
mv Cours/TP/Notes/copie.txt Cours/
mv Cours/copie.txt Cours/final.txt
ls -R Cours
Astuce : dressez un tableau d'équivalence DOS ↔ UNIX (dir/ls, copy/cp, move/mv, del/rm, type/cat, cls/clear). Il vous resservira toute votre carrière.Tip: build a DOS ↔ UNIX equivalence table (dir/ls, copy/cp, move/mv, del/rm, type/cat, cls/clear). It will serve you your whole career.

Matériel / prérequisMaterials / prerequisites

Imprimante réseau (ou simulée), accès au réseau, droits administrateur.A network printer (or simulated), network access, administrator rights.

Étapes guidéesGuided steps

  1. Préparation : récupérer le manuel + le pilote officiel sur le site du constructeur.Preparation: get the manual + official driver from the maker's site.
  2. Sauvegarde : créer un point de restauration.Backup: create a restore point.

⏱ 1 h 30BinômePairsPratiqueHands-on

ObjectifObjective

Appliquer la démarche de diagnostic en 5 temps pour identifier et résoudre un problème de périphérique.Apply the 5-step diagnostic process to identify and fix a peripheral problem.

Étapes guidéesGuided steps

  1. Provoquer/observer un cas : pilote désinstallé, câble débranché, ou périphérique non détecté.Create/observe a case: uninstalled driver, unplugged cable, or undetected device.

Étapes guidéesGuided steps

  1. Lire les prérequis sur code.visualstudio.com puis vérifier sa config (msinfo32 / lscpu).Read the requirements on code.visualstudio.com then check your specs (msinfo32 / lscpu).
